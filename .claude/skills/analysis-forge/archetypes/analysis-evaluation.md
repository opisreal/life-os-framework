# Archetype：分析/评估型（v0.1 唯一 archetype）

> 引擎（流水线）领域无关；本文件承载**类型差异**——什么样的制品、拆什么样的 claim、默认扇出哪些 lens、产物长什么样。生成型（文风模仿）不属本 archetype，触发即拒并说明留 v0.3。

## 适用制品类型

| 制品 | 逆向目标 | 生成物 |
|---|---|---|
| 分析结论（市场报告 / 投研 / 诊断） | 分析链 + 数据 + 框架 | 分析型 skill（可重复产出同类分析） |
| 演讲 / 访谈 | 思维方式 + 知识储备 + 见识结构 | 评估型 skill（"复盘这个人的思维"） |
| 带论证的结论 / 文章 | 到达结论的证据链 | 评估型 skill（证据链审计） |

判别不适用（→ 可行性门拒绝或降级）：
- 纯叙事 / 抒情文本，无可拆 claim → 拒绝
- 制品只有结论没有任何论证痕迹 → 降级为"仅分析模板"，要求用户补材料
- 用户要的是"模仿这个文风写东西" → 生成型，v0.1 不做

## Claim 类型表

| 类型 | 特征 | 例 |
|---|---|---|
| 状态判定 | 给当前局面定性（regime） | "超卖反抽窗口，非单边上行" |
| 价位/阈值 | 关键数值位 + 其意义 | "1550 = 多空分界" |
| 触发 | if-then 条件化判断 | "站上 1640 则逻辑改变" |
| 归因 | 解释为什么发生 | "下跌 = 清算链 + 情绪弱 + 资金流出" |
| 预测 | 对未来的有条件断言 | "失守则看 1486" |
| 知识前提 | 未明说但支撑论证的背景知识 | "清算密集区会放大波动" |

## 默认 lens 模式（按制品类型）

分析型（市场/投研类）：
- 数据 lens：每类独立数据输入一个 lens（价格结构 / 情绪 / 资金流 / 持仓杠杆 / 链上）
- 结构 lens：关键位识别（多源汇合打分）
- 联动 lens：相关市场/基准的牵引（原制品常见盲区，缺口诊断重点查）

评估型（演讲/思维复盘类）：
- 论证结构 lens：claim 间依赖是否闭合、有无跳步
- 知识储备 lens：知识前提清单 + 各自的领域深度要求
- 证据质量 lens：一手/二手/转述分级，可核验性
- 盲区 lens：同领域专家会问而制品没答的问题

lens 数量默认 3-5 个；超过 5 个需在门 1 说明理由。

## 生成 skill 的输出契约

生成的分析/评估 skill 报告必须含：
1. **状态判定**（regime / 总评）+ 校准置信度（高/中/低 + 三因子理由）
2. **分维度发现**（每个 lens 一节，标数据来源与时间戳）
3. **对抗证伪记录**（skeptic 的最强反驳 + 为何不改变/改变了结论）
4. **触发条件 / 失效条件**（可量化规则，不用"放量""企稳"等模糊词单独成条）
5. **数据缺失声明**（拉不到的源显式列出，不许静默省略）

## 禁止事项

- ❌ 生成的 skill 写死实例值（币种/价位/人名/日期）
- ❌ lens 引用未经 sample_passed 的数据源
- ❌ 输出数字化评分（"准确率 85%"）——置信度只用三档 + 理由
- ❌ 把二手转述数据当一手数据用（必须标"二手，需独立核验"并降权）
- ❌ 评估型 skill 对人做人格/道德判断——只评估论证质量与知识结构
